Преимущества

Без лишних трат

Оплата только за фактически потребленные ресурсы. Мы отказались от учета входящего трафика и вы платите только за хранение, запросы к API и скачивание данных из хранилища.

Высокая надежность

Автоматическое создание 3 реплик данных на отдельных независимых серверах. Надежность на уровне Tier III, SLA гарантирует доступность 99,98% времени.

Масштабируемость

Вне зависимости от объема данных и уровня пиковой нагрузки сохраняется быстрый доступ к контенту. Количество и объем файлов не ограничены.

Простая интеграция

Подключение собственного либо стороннего приложения через API OpenStack Swift. Управление пользователями, доменами, сертификатами через API. Ряд готовых плагинов и интеграций с CMS, CRM и другими сервисами.

Удобный доступ

Весь функционал доступен как через API, так и через панель управления: загрузка и скачивание файлов, разграничение прав доступа, привязка доменов, сбор статистики.

Возможности облачного хранилища

Статические данные

Снижайте нагрузку на сайт и освобождайте вычислительные мощности. Сохраняйте файлы любого формата и размера — контент будет доступен при любых нагрузках без ограничений по скорости раздачи.

  • CDN — подключайте сеть доставки контента для более эффективной раздачи файлов посетителям вашего сайта
  • Кэширование — настраивайте параметры HTTP-заголовков на уровне контейнера и файла
  • Лимиты — установите ограничения на общий объем информации и время хранения файлов
  • Контент — раздавайте данные с помощью публичных контейнеров
  • Домены — привязывайте домены 2-го уровня для статических сайтов и 3-го уровня для отдельных ресурсов к публичным контейнерам
  • CMS — подключите проект к Bitrix и Wordpress
  • Статический сайт — настройте индексную страницу и ошибку 404
  • Защищенное соединение— используйте прямой трафик хранилища и CDN, который шифруется SSL-сертификатами

Резервные копии

Храните резервные копии без ограничений по объему. Данные реплицируются на три независимых сервера.

  • Приватные контейнеры — храните данные с доступом по логину и паролю
  • Версионирование — сохраняйте перезаписанные файлы в отдельном контейнере с указанием времени, обращайтесь к нужной копии в любой момент
  • Автоматическое удаление — настраивайте срок хранения файлов, чтобы автоматически удалять неактуальные копии и не платить за хранение ненужных бэкапов

Документы

Обменивайтесь файлами с коллегами и удаленными заказчиками. Оформляйте и настраивайте хранилище как вам удобно с помощью API.

  • Управление доступом — создавайте неограниченное количество пользователей с правами доступа к определенным контейнерам
  • Легкая загрузка — создавайте ссылки для загрузки файлов без регистрации в сервисе
  • Веб-листинг контейнера — список файлов и папок отображается в браузере, вы можете подключить собственные стили. Просматривайте список файлов и папок в браузере; подключите свои стили, чтобы кастомизировать проект
  • Удобные фотогалереи — разрешите пользователям просматривать и скачивать фотографии. Галерея адаптирована для мобильных устройств

Как создать архив tar.gz в любой версии линукс

Формат архива tar.gz является стандартом для linux систем и используется повсеместно в качестве архивов, репозиториев и установочных модулей.

В некоторых случаях есть необходимость самостоятельно создать архив этого формата.

Как создать архив .tar.gz

Сама команда tar ничего не компрессирует а всего лишь «сливает» файлы и/или директории в один общий файл. Последующая компрессия выполняется посредством архиватора gzip.

Ниже приведены основные примеры использования команды tar:

tar -cvf file.tar /full/path — создать .tar
tar -czvf file.tar.gz /full/path — создать .tar.gz (архив)

Синтаксис для работы с командой tar:

tar [-ключи] [название архива] [путь, что запаковать]

Значения ключей:

  • с = «create» : создать файл архива
  • v = «verbose» : выводить информацию в процессе выполнения
  • f = «file» : использовать имя файла архива указаное после ключей. Если не указать ключ f то команда будет использовать настройки по умолчанию либо выведет результат прямо в консоль
  • z = «gzip» : запаковать файл при помощи gzip

 

Как открыть (распаковать) .tar

Чтобы распаковать запаковыный .tar.gz:

tar -xvf file.tar.gz

Синтаксис:

tar [-ключи] [название архива]

Значения ключей:

  • x = «eXtract» : извлечь файлы
  • v = «verbose» : выводить информацию в процессе выполнения
  • f = «file» : использовать имя файла архива для распаковки указаное после ключей
