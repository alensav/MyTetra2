Решение проблем OpenCart. Как решить проблему, если ваша админ панель перенаправляет Вас на страницу установки

Иногда, когда Вы пытаетесь войти в админ панель сайта Opencart, Вы увидите лишь страницу установки. Причиной этого, скорее всего, является файл /admin/config.php. Либо этому файлу присвоены неправильные привилегии, либо он пустой. Для того чтобы устранить проблему, следуйте таким инструкциям:

I. Неправильные привилегии

  1. Откройте файлы вашего сайта на сервере (используя Панель управления хостингом или клиент FTP) и перейдите в папку ‘admin’.
  2. Найдите файл ‘config.php’. Измените привилегии на 644 или 666.

II. Пустой файл /admin/config.php

  1. Откройте файлы сайта на сервере (используя Панель управления хостингом или клиент FTP) и откройте папку ‘admin’.
  2. Найдите файл ‘config.php’ и откройте его для просмотра содержимого. Скорее всего, этот файл будет пустой или его содержимое будет неполным.
  3. Вам нужно добавить нужное содержимое в файл. Этот файл должен содержать ссылки сайта и информацию о базе данных. Вы можете использовать образец файла config.php. Для того чтобы файл подходил для вашего магазина, необходимо внести следующие изменения:
    1. Откройте файл-образец в любом редакторе HTML и измените все упоминания your_website_url.here на фактический адрес вашего сайта (без www).

      2. Обратите внимание: если шаблон установлен в подпапку, её название нужно также указать для всех папок:

      define('HTTP_SERVER', 'http://your_websiteurl.here/subfolder/admin/');

    2. Папки с файлами также указаны в файле. Поэтому, все упоминания full_server_files_directory нужно заменить на фактический путь к файлам вашего сайта на сервере. Откройте файл config.php в корневой папке вашего сайта, для того чтобы скопировать правильный путь (например, full_server_files_directory нужно заменить на home3/mgreen/public_html/judy).
    3. Укажите информацию о базе данных (драйвер БД, название хоста БД, название БД, пользователя БД и пароль БД). Вы можете найти эти данные в файле ‘config.php’ в корневой папке вашего сайта.
  4. Обновите файл /admin/config.php, после того как все остальные файлы будут обновлены.
  5. Вернитесь в корневую папку вашего сайта. Найдите папку ‘Install’. Её нужно переименовать или удалить, для того чтобы избежать повторного запуска установки.

Если Вы проверите вашу админ панель сейчас, она должна работать корректно. Теперь Вы знаете, как решить проблему, если ваша админ панель перенаправляет Вас на страницу установки в OpenCart.
