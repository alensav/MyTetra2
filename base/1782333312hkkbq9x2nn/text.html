📱 ЧАСТЬ 3. Если MyTetroid не видит новые записи

Проблема: Вы запустили ~/mt-sync.sh, но в MyTetroid не появились новые записи.

Решение: Перезапустите MyTetroid.

  1. Полностью закройте MyTetroid (свайпните его из списка приложений).
  2. Откройте MyTetroid снова.
  3. Записи должны появиться.

Почему: MyTetroid обновляет индекс только при запуске. Это особенность мобильного приложения.

⚠️ ВАЖНЫЕ ПРАВИЛА

  1. Никогда не используйте --delete в rsync при копировании из GitHub в MyTetroid. Только при копировании из MyTetroid в GitHub.
  2. Всегда перезапускайте MyTetroid после ~/mt-sync.sh, чтобы он увидел новые записи.
  3. Закрывайте MyTetra через меню "Выход", чтобы он успел обновить индекс.
  4. Не работайте одновременно в MyTetra и MyTetroid с одной и той же записью — будут конфликты.

🎯 Итог

Теперь у вас есть полностью работающая система синхронизации между Astra Linux и Android через GitHub.

Ключевые моменты для запоминания:

  1. ~/mt-sync.sh — ваша главная команда в Termux.
  2. Перезапуск MyTetroid — обязателен после синхронизации.
  3. git pull на Astra — чтобы получить записи с телефона.

Сохраните эту инструкцию — она пригодится каждый день! Если что-то пойдёт не так, просто выполните полный цикл (Ситуация 3) — это восстановит синхронизацию.
