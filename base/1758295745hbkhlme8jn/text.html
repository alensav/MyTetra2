Что дальше? Вот полезные команды для git-bug v0.10.1:


КОМАНДА

ОПИСАНИЕ

git-bug add

Создать новый баг

git-bug ls

Список багов

git-bug show <ID>

Показать детали бага

git-bug comment <ID>

Добавить комментарий

git-bug status <ID> <status>

Изменить статус (например,closed)

git-bug label <ID> add <label>

Добавить метку

git-bug user ls

Посмотреть доступные идентичности

git-bug webui

Запустить веб-интерфейс (если поддерживается)

git-bug termui

Запустить терминальный интерфейс


Пример: добавим комментарий

bash

1

git-bug comment e630

→ В редакторе:

1

Первый комментарий — всё работает!

bash

bash

1

Совет: синхронизация с другими

Так как git-bug работает через Git — просто делайте:

bash


git add .git/git-bug/

git commit -m "Добавлены баги"

git push

→ Другие участники увидят баги после git pull.
