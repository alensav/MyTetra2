Выход → DigiSpark (no USB bootloader) ??

Qwen3-Next-80B-A3B

Отличный и очень важный вопрос — вы затронули ключевую тонкость, которая сбивает с толку даже опытных пользователей.


❓ Что означает:

"Выход → DigiSpark (no USB bootloader)"
...и почему он используется при записи загрузчика на ATtiny85?


✅ Краткий ответ:

“DigiSpark (no USB bootloader)” — это не название платы, а настройка загрузчика в библиотеке ATTinyCore.
Она говорит Arduino IDE:
«Запиши на ATtiny85 загрузчик Micronucleus, который позволяет программировать его по USB, как DigiSpark — но без предустановленного USB-загрузчика».
То есть — это способ записать именно тот самый загрузчик, который делает ATtiny85 совместимым с USB-программированием.


Подробное объяснение

Что такое Micronucleus?

Это легковесный USB-загрузчик для ATtiny85, разработанный специально для миниатюрных плат вроде DigiSpark.
Он:

  • Использует USB-интерфейс через V-USB (программную эмуляцию USB)
  • Позволяет загружать скетчи напрямую через Micro-USB, без внешнего программатора
  • Занимает ~2 кБ флеш-памяти
  • Автоматически запускается при включении питания (или сбросе)

Без этого загрузчика ATtiny85 не отвечает по USB — он просто молчит, как "голый" чип.


Почему в меню стоит "DigiSpark (no USB bootloader)"?

Потому что в ATTinyCore есть два режима записи загрузчика:


ОПЦИЯ

НАЗНАЧЕНИЕ

DigiSpark (no USB bootloader)

⚠️Используйте ЭТОприпервой записи загрузчиканаголый ATtiny85— записываетбазовый Micronucleus(тот, что работает на 16.5 МГц)

DigiSpark (with USB bootloader)

✅ Используйтетолько после того, как загрузчик уже записан— для обновления или перезаписи (обычно не нужен)

❗ Это название может вводить в заблуждение — ведь вы не подключаете настоящий DigiSpark!
Но внутри ATTinyCore эта опция — единственный способ записать Micronucleus.


Процесс записи загрузчика: Почему именно так?

Вот правильная последовательность:

🟡 Шаг 1: Первичная прошивка — записываем базовый загрузчик

  • Частота: 8 MHz (internal)
  • Выход: DigiSpark (no USB bootloader)
  • Программатор: Arduino as ISP

→ Arduino IDE записывает файл:
.../bootloaders/micronucleus/t85_entry_on_power_on_no_pullup_fast_exit_on_no_USB.hex

✅ Этот загрузчик:

  • Устанавливает фьюзы на 8 МГц
  • Записывает первую версию Micronucleus, готовую к обновлению
  • Не использует USB пока — но готов к нему

После этого ATtiny85 ещё не умеет работать по USB — но теперь он может быть обновлён до полноценного USB-режима.


🟢 Шаг 2: Обновление до USB-режима — пишем финальный загрузчик

  • Частота: 16.5 MHz (internal) ← обязательно!
  • Выход: DigiSpark (no USB bootloader) ← опять же!
  • Программатор: Arduino as ISP

→ Arduino IDE записывает файл:
.../bootloaders/micronucleus/upgrade-t85_entry_on_power_on_no_pullup_fast_exit_on_no_USB.hex

✅ Этот загрузчик:

  • Переходит на 16.5 МГц (реальная частота, которую поддерживает V-USB)
  • Активирует USB-интерфейс
  • Теперь ATtiny85 полностью превращается в DigiSpark

⚠️ Важно:
Вы НЕ выбираете «DigiSpark (with USB bootloader)» — потому что он предназначен только для обновления уже работающего загрузчика, а не для первоначальной установки!

Это не идеально — но работает.

Выход: DigiSpark (no USB bootloader) ← опять же! Где мы это видим и что настраиваем?

Отличный уточняющий вопрос — вы находитесь на самом ключевом этапе прошивки ATtiny85, и именно здесь многие теряются. Давайте разберёмся до мелочей:

Где именно находится эта опция в Arduino IDE?
✅ Что она настраивает?
Как её выбрать правильно?


✅ Ответ кратко:

Опция DigiSpark (no USB bootloader) находится в меню Инструменты → Выход (или Board) в Arduino IDE — после того, как вы выбрали плату ATtiny85.
Она не выбирает физическую плату, а выбирает тип загрузчика, который будет записан на ATtiny85 — именно тот, что позволяет ему работать по USB как DigiSpark.


Шаг 1: Где найти эту опцию? (Пошагово с картинками в тексте)

1. Подключите Arduino Mega/Uno к ПК → запустите Arduino IDE

2. Убедитесь, что установлено ATTinyCore

  • Инструменты → Платы → Менеджер плат
  • Найдите ATTinyCore → установите последнюю версию (1.5.2+)

⚠️ Не используйте старую библиотеку damellis/attiny — она не поддерживает Micronucleus!

3. Выберите плату ATtiny85

  • Перейдите: Инструменты → Плата → ATtiny85

✅ После этого появится новое подменю — "Выход" (Variant)

4. Теперь найдите опцию:

Инструменты → Выход → DigiSpark (no USB bootloader)

🔁 Когда и как использовать эту опцию? (Правильная последовательность)

Вот единственно правильная цепочка действий:

🟡 Шаг 1: Первоначальная прошивка (на голом ATtiny85)

  • Плата: ATtiny85
  • Частота: 8 MHz (internal)
  • Программатор: Arduino as ISP
  • Выход: DigiSpark (no USB bootloader) ← ✅ ВЫБИРАЕТЕ ЭТО!

→ Записывается базовый загрузчик
→ Фьюзы настраиваются на 8 МГц
→ USB пока не работает — но чип теперь готов к обновлению

✅ После этого светодиод на ATtiny85 может мигать — это признак успешной записи.


🟢 Шаг 2: Активация USB (ключевой момент!)

  • Плата: ATtiny85 (остаётся та же!)
  • Частота: 16.5 MHz (internal) ← ВАЖНО! Измените!
  • Программатор: Arduino as ISP (остаётся)
  • Выход: DigiSpark (no USB bootloader) ← ✅ ОПЯТЬ ЭТО!

→ Arduino IDE перезапишет загрузчик на обновлённую версию, которая:

  • Переключает тактовую частоту на 16.5 МГц (реально необходимая для V-USB)
  • Активирует USB-интерфейс
  • Теперь ATtiny85 полностью превращается в DigiSpark

💡 Это не новая опция — это та же самая, но при другой частоте!
Именно поэтому никогда не меняйте “Выход” — только частоту.


🔵 Шаг 3: Загрузка скетча (после успешной прошивки загрузчика)

  • Плата: ATtiny85
  • Частота: 16.5 MHz (internal)
  • Программатор: Arduino as ISP → Но!
  • Выход: DigiSpark (no USB bootloader) ← ВСЁ ЕЩЁ ЭТО!

❗ Почему не "USBasp"?
Потому что теперь ATtiny85 сам становится USB-устройством — он не нуждается в программаторе!
Но Arduino IDE всё ещё требует выбрать "Arduino as ISP", чтобы начать процесс загрузки.

✅ После нажатия "Загрузить":

  • Arduino IDE ждёт 5–7 секунд
  • Подключите HW-260 (или ATtiny85) к USB
  • Через 1–2 секунды — скетч загрузится!

💡 Это работает так же, как DigiSpark — вы просто вставляете плату в USB, и она сама принимает код.
