Посмотрите, какие снимки уже созданы

bash

ls -la /mnt/timeshift/timeshift/snapshots/

Вы увидите папку с датой и временем — это ваш первый снимок.

  • Домашняя папка исключена (сохраняется только система)

Что теперь делать с Timeshift

  1. Ручные снимки — делайте перед важными действиями:

    2. bash

    sudo timeshift --create --comments "Перед обновлением"

    Или через графический интерфейс: sudo timeshift-gtk → кнопка «Создать»

  2. Проверьте расписание — откройте Timeshift (GTK), зайдите в настройки (шестеренка) → вкладка «Расписание» и убедитесь, что настроено:
    • Ежедневно: 3–5 снимков
    • Еженедельно: 2–3 снимка
  3. Восстановление — если система сломается:
  • Если загружается: sudo timeshift-gtk → выбрать снимок → «Восстановить»
  • Если не загружается: загрузитесь с Live USB, установите Timeshift и восстановите снимок с внешнего диска

Дополнительно: настройте автоматическое монтирование диска

Убедитесь, что после перезагрузки диск автоматически примонтируется. Проверьте:

bash

cat /etc/fstab | grep timeshift

Если строки нет — добавьте (это важно для автоматических снимков по расписанию):

bash

# Узнайте UUID

sudo blkid /dev/sdc2


# Добавьте в /etc/fstab

sudo nano /etc/fstab

Добавьте строку:

text

UUID=ваш-uuid-от-blkid /mnt/timeshift ext4 defaults,nofail 0 2

Сохраните (Ctrl+O, Ctrl+X).

Что будет после заполнения диска?

Отличный вопрос! Timeshift имеет встроенные механизмы для автоматического управления местом на диске.

Как Timeshift управляет заполнением диска

1. Автоматическое удаление старых снимков

Timeshift автоматически удаляет самые старые снимки, когда:

  • Достигнуто максимальное количество снимков, которое вы задали в расписании
  • Или когда свободного места становится мало

В ваших настройках (если вы следовали рекомендациям):

  • Ежедневные: хранить 3–5 снимков
  • Еженедельные: хранить 2–3 снимка
  • Ежемесячные: хранить 1 снимок

Когда создается новый снимок, Timeshift удаляет самый старый, если лимит превышен.

2. Проверьте текущие настройки

bash

sudo timeshift --list

Или в графическом интерфейсе: sudo timeshift-gtk → Настройки → Расписание

3. Что произойдет, если место всё же закончится

Если свободного места станет критически мало (менее 1 ГБ):

  1. Timeshift выдаст предупреждение при попытке создать новый снимок
  2. Создание снимка будет заблокировано до тех пор, пока вы не освободите место
  3. Существующие снимки не будут повреждены — они останутся нетронутыми

Рекомендации по управлению местом

Ваш текущий объем:

  • Раздел /dev/sdc2: 283 ГБ
  • Занято снимками: 30 ГБ
  • Свободно: ~253 ГБ

Это очень хороший запас. При таком объеме вы сможете хранить:

  • 8–10 полноценных снимков (если каждый по 25–30 ГБ)

Мониторинг свободного места

Периодически проверяйте:

bash

df -h /mnt/timeshift

Вывод будет примерно такой:

text

Filesystem Size Used Avail Use% Mounted on

/dev/sdc2 278G 30G 234G 12% /mnt/timeshift

Что делать, если место подходит к концу

Вариант 1: Вручную удалить старые снимки

bash

# Посмотреть все снимки

sudo timeshift --list


# Удалить конкретный снимк (по номеру или дате)

sudo timeshift --delete --snapshot "2026-03-20_12-00-00"

Или в графическом интерфейсе: выбрать снимок → кнопка «Удалить».

Вариант 2: Изменить настройки хранения

Уменьшите количество хранимых снимков:

  • Ежедневные: с 5 до 3
  • Еженедельные: с 3 до 2
  • Ежемесячные: отключите

Вариант 3: Очистить кэш Timeshift (редко помогает, но бывает)

bash

sudo rm -rf /mnt/timeshift/timeshift/snapshots-boot/*

Важное замечание

Timeshift никогда не удалит последний снимок, даже если место совсем закончится. Это защита от полной потери точек восстановления.

Если место станет критически малым, вы увидите сообщение:

Not enough free space — и снимок не будет создан, пока вы не удалите старые вручную или не увеличите раздел.

  • При необходимости всегда можно удалить ненужные снимки вручную

Всё настроено правильно и будет работать автоматически.
