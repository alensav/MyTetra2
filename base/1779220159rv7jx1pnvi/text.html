Пояснение

mount /mnt/mailru

❌ can't find in /etc/fstab

Работает только если точка прописана в /etc/fstab. Пока не добавили — монтируем полной командой.

df -h /mnt/mailru

✅ Показывает 164 ГБ

Диск уже подключён (с предыдущего сеанса), поэтому df работает.

cp ~/document.pdf ...

❌ Файл не найден

Это был пример команды. Файла document.pdf в вашей домашней папке нет — подставьте свой файл.

umount /mnt/mailru

❌ Операция не позволена

Размонтирование требует прав root, если монтировалось через sudo.

sudo umount -l /mnt/mailru

✅ Успешно

Правильный способ размонтирования.

🚀 Шпаргалка на каждый день

bash

# 🔗 Подключить облако:

sudo mount -t davfs https://webdav.cloud.mail.ru/ /mnt/mailru \

  -o uid=alensavas,gid=alensavas,file_mode=0644,dir_mode=0755

# 📁 Скопировать файл в облако:

cp ~/важфайл.doc /mnt/mailru/Backups/

# 📥 Скачать из облака:

cp /mnt/mailru/Backups/важфайл.doc ~/Загрузки/

# 🔍 Проверить, что диск подключён:

mount | grep mailru

df -h /mnt/mailru

# 🔌 Размонтировать (когда закончили):

sudo umount -l /mnt/mailru

💡 Важно: благодаря delay_upload 1 команда cp завершается мгновенно, но файл может появиться в веб-интерфейсе Mail.ru через 10–30 секунд (пока идёт фоновая загрузка). Это нормально!

Итоговая шпаргалка для повседневной работы

bash
# 🔗 ПОДКЛЮЧИТЬ (после настройки fstab):
mount /mnt/mailru
# или без fstab:
sudo mount -t davfs https://webdav.cloud.mail.ru/ /mnt/mailru -o uid=alensavas,gid=alensavas

# 📁 РАБОТАТЬ С ФАЙЛАМИ:
cp ~/document.pdf /mnt/mailru/Backups/
rsync -av --progress ~/project/ /mnt/mailru/Backups/project/
ls -lh /mnt/mailru/

# 🔌 ОТКЛЮЧИТЬ:
umount /mnt/mailru
# или если не получается:
sudo umount -l /mnt/mailru

# ✅ ПРОВЕРИТЬ СТАТУС:
mountpoint -q /mnt/mailru && echo "🟢 Подключено" || echo "🔴 Отключено"
df -h /mnt/mailru
